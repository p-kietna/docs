Great question! If you're looking to run `async/await` inside a loop **safely and sequentially**, here are the loop types that work ‚Äî and those that don‚Äôt.

### ‚úÖ Loops That Support `await`

These loops **can run `await` properly**, meaning each iteration waits for the previous one to finish:

#### 1. **`for...of` loop**
Best for sequential async operations.

```ts
for (const item of items) {
  await doSomethingAsync(item);
}
```

#### 2. **Classic `for` loop**
Also works fine with `await`.

```ts
for (let i = 0; i < items.length; i++) {
  await doSomethingAsync(items[i]);
}
```

#### 3. **`for await...of` loop**
Used for **async iterables** like streams or paginated APIs.

```ts
for await (const item of asyncIterable) {
  // item is resolved from a promise
}
```

---

### ‚ùå Loops That Don‚Äôt Work with `await` (without extra care)

These loops **don‚Äôt wait** for `await` to finish before continuing:

- `Array.prototype.forEach`
- `map`
- `reduce` (unless chained properly)
- `filter`, `some`, `every`

```ts
items.forEach(async item => {
  await doSomethingAsync(item); // ‚ùå Doesn't wait
});
```
